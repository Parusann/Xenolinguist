# Xenolinguist — Full Code Review (2026-06-13)

## Method & scope

A multi-agent review of the **`origin/main`** tree (content-identical to the `voice/ipa-extras` branch, which contains the full TTS + Whisper-STT + phonetic/IPA voice subsystem). 13 feature-area reviewers + 4 cross-cutting reviewers (security, resilience, data-integrity, type/contract) read the code; **every finding was then adversarially verified** against the source. Of 152 raw findings, **129 survived verification**. This report records what was fixed this session and the verified recommendations that remain.

Work was done on branch `review/full-audit` in the `Xenolinguist-impl` worktree.

## Update — all recommendations applied

Following the initial 6 fixes, **all remaining verified recommendations were implemented** on the same branch. Post-implementation suites are green: **server 51 passed / 3 skipped, client 15 passed, client build + electron bundle clean** (server `tsc` still reports only the pre-existing `?query` cache-bust import errors in test files — not a regression; the server ships via the esbuild bundle).

Highlights of what was applied across the codebase: atomic profile writes + in-process index lock + self-heal + validate-on-read (`profile-store.ts`); Ollama/whisper/espeak request timeouts; per-route JSON body limits + dev-only CORS; whitelist/validation on all request bodies (`pickProfileData`, `isValidMessages`, WAV magic-byte checks); `AbortController` wired through `streamAI`/AIChat; object-URL & AudioContext leak fixes; segment-window playback in `AudioPlayer`; true-restore undo (`addDictionaryEntryRaw`); sandbox number-system population + POS normalization + in-app "Play Again"; dialog/aria semantics + aria-live toasts; CSV quote-escaping + import shape validation; receive-only Electron IPC bridge (`ollama:offline`/`pull-progress`); shared `ProfileIndex` type; dead-code removals (`getModels`, `apiFetch` adoption, `AIRequest`, `electron-ollama` dep, dead base64 catches); marketing copy/links fixed.

**Three items were intentionally deferred** (rationale, not omission):
- **Sandbox duplicate-word keying (#47)** — keying the sandbox state maps by array-id instead of alien word is a structural change across a ~1150-line file; risk outweighs the value (generated conlangs rarely contain duplicate alien words).
- **NumberDecoder per-keystroke local state (#79)** — the network concern is already mitigated by the existing 500 ms persist debounce; a local-mirror refactor of the controlled grid risks sync bugs for negligible gain.
- **Dev auto-start `argv[1]` heuristic (#91)** — a documented dev-only convenience with an existing `XENO_START=1` escape hatch.

`#109` (AI output is advisory) is addressed in `docs/FEATURES.md` (§16 Intentional Behaviors).

## 2. Recommended — Correctness (verified, not yet applied)

Highest-value first. None block normal use; most are edge-case/robustness.

**Persistence & data integrity (`profile-store.ts`)**
- Non-atomic JSON writes can corrupt `profiles.json` / a profile file on interruption — write to a temp file then `rename` (atomic on same volume).
- Lost-update race on the shared index (read-modify-write with no lock) — serialize index mutations through an in-process async lock.
- `create()` writes profile file then index non-transactionally; a failure between desyncs the index — reconcile `list()` against the directory, or write atomically.
- `update()` silently no-ops the index when the profile id is missing from `profiles.json` — self-heal by inserting the row.
- `get()` casts parsed JSON to `LanguageProfile` with no validation — merge over `createDefaultProfile()` to backfill missing fields.

**AI / resilience**
- No request timeout on any Ollama call (`ai-service.ts`) — add `AbortController` with `OLLAMA_TIMEOUT_MS`.
- Spawned whisper/espeak have no process timeout (`stt-whisper.ts`, `tts-espeak.ts`) — arm a `setTimeout` that kills the child and rejects.
- `profiles` router async handlers are unguarded (`routes/profiles.ts`) — wrap in try/catch→`next(err)` (the pattern `ai.ts` already uses).
- `startServer()` has no listen-error handler — `EADDRINUSE` hangs instead of rejecting (`server/src/index.ts`).
- No global `unhandledRejection`/`uncaughtException` handlers in the server-entry or Electron main.

**Client state & UX**
- `createProfile` failure leaves the submit button stuck on "Creating…" forever (`ProfileSetup.tsx`) — try/catch + reset.
- `loadProfile` / several landing fetches parse error responses as profile data (no `res.ok` check) — route through `apiFetch` or check `res.ok`.
- Saved-profile click calls `loadProfile` without await/catch → unhandled rejection (`LandingScreen.tsx:335`).
- Decoder AI handlers (Number/Vocab/Grammar/Translation) have no try/catch around `runTask` → unhandled rejection, no user-facing error.
- Audio clip persisted in profile state even when `POST /api/audio/upload` fails (`SampleInput.tsx`) — await upload, persist only on success.
- `removeSample` orphans the server-side audio file — also remove the clip + `DELETE /api/audio/:id`.
- Object-URL / `AudioContext` leaks: `pendingAudio` URL not revoked on add/re-upload; `AudioContext` leaked when `AudioRecorder` unmounts mid-recording or `decodeAudioData` throws.
- Re-transcribe overwrites user-typed `phonetic_notes` with no merge/confirmation.
- Transcript-linked dictionary words are never written back to `AudioSegment.dictionary_entry_id` (link is lost).
- Decode-view / translation tokenization is whitespace-only → punctuated/inflected words miss the dictionary.
- Sandbox-decoded numbers never populate `number_system.mappings`, so the Dashboard under-reports Numbers.
- CSV export doesn't escape embedded double-quotes — corrupts the file for quoted meanings/contexts.
- macOS re-activate forks a second orphaned server process (`electron/main.ts`).
- Conlang POS strings drift from the `PartOfSpeech` enum via `pos as any` — normalize on persist.
- `port()` treats blank `PORT` as `0` (silent OS-assigned) instead of the `3001` default.
- Renderer-targeted IPC events (`ollama:offline`, `ollama:pull-progress`) are unreceivable — preload exposes no bridge and no client listens.

## 3. Recommended — Security (verified, all low / defense-in-depth)

- No validation of incoming profile bodies on POST/PUT — arbitrary/oversized data persisted verbatim; normalize to a known-field whitelist in `create()`/`update()`.
- 50 MB JSON body limit + base64 audio, no rate limiting → memory-amplification DoS; lower the global limit and/or apply per-route limits.
- Audio upload trusts `mimeType` for the extension and never validates content — check leading magic bytes (`RIFF…WAVE` / EBML).
- CORS origin hard-coded to the Vite dev origin in all builds (moot in prod where renderer + API are same-origin) — gate on build mode or drop `cors()` in production.
- espeak `voice`/`text`/`phonemes` length is uncapped — cap to stay under OS argv limits (partly addressed by fix #6; a length cap is still advisable).

## 4. Recommended — Improvements & cleanups (selected)

**Improvements**
- Thread an `AbortController`/`signal` through `streamAI` so closing the AI chat actually cancels server-side generation (today it keeps running).
- `streamAI` doesn't check `res.ok` before reading the SSE body — non-2xx responses are read as a stream.
- Accessibility: custom modal overlays (CommandPalette, ShortcutsHelp) lack `role="dialog"`/`aria-modal`/focus trap; toasts lack an `aria-live` region.
- IPA/STT: validate (or resample) the WAV sample rate — non-16 kHz audio yields acoustically wrong phones with no warning; map bad-input WAVs to `400` rather than `503`.
- Sandbox "Play Again" does a full `window.location.reload()` — replace with in-app state reset.
- Translation AI prompt omits samples (vocab/grammar prompts include them).

**Cleanups (low-risk)**
- Dead code: `AIService.getModels()`, the exported `apiFetch` (unused), unreachable base64 `try/catch` in `stt.ts`, `AIRequest.task/stream` fields, the hardcoded duplicate "Audio recording" `<option>`.
- Dead dependency: `electron-ollama` is declared but never imported — remove to slim the installer.
- Duplication: `OLLAMA_BASE_URL` defined twice (add a `config.ts` accessor); `ProfileIndex` type duplicated (export `Pick<LanguageProfile,…>` from `shared/types.ts`); the CTC decode loop is copied into `verify-ipa.mjs` (import the compiled decoder to prevent drift); `MIN_LANGUAGE_PROB` duplicated in `verify-stt.mjs`.
- `createApp()` runs eagerly at import time (`app.ts:50`) — reads env before tests can set it; drop the eager singleton.
- `.tts-btn` CSS class is referenced but never defined.

**Docs/UX accuracy**
- Marketing demo advertises `llama3.1:70b`, but the app defaults to `gemma4:e4b` — update the copy.
- Marketing footer/CTA has dead links / non-functional buttons.
- Clarify in-UI that the four phase AI passes are advisory and not auto-applied.
